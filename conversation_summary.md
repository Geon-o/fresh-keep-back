# FreshKeep 프로젝트 작업 및 대화 내용 요약 (2026-06-26)

본 문서는 기기 변경/재설치 시 데이터 복구 로직 고도화 및 AI 식재료 보관 팁 조회 기능의 신뢰성 개선을 위해 진행한 대화 및 작업 내역을 요약한 기록입니다.

## 1. 복구(Restore) 시 임시 계정 완전 삭제 및 Unique 충돌 해결

### 1.1. 배경 및 문제점
* 사용자가 앱 재설치 시 신규 임시 계정이 생성되고, 이후 기존 백업 키로 복구(Restore)를 시도할 때 기존 계정에 현재 기기 UUID를 할당해야 합니다.
* 이때 기존 임시 계정의 `device_uuid`가 Unique 제약 조건에 걸려 충돌이 발생하거나, 단순히 `null`로 남겨두면 쓰지 않는 유저 계정(쓰레기 데이터)이 데이터베이스에 무한 누적되어 성능 및 자원 효율이 뒤떨어지는 문제가 지적되었습니다.

### 1.2. 개선 및 버그 해결 내용
* **임시 계정 및 연관 데이터 연쇄 소거 (Cascade Delete)**:
  * 복구 성공 시, 현재 기기 UUID를 점유하던 임시 사용자(`tempUser`) 및 그에 매핑되어 있던 냉장고 멤버십 관계(`FridgeMember`), 그리고 멤버가 남아있지 않은 임시 독점 냉장고(`Fridge`)를 DB에서 완전히 삭제(Delete)하도록 백엔드 로직을 변경했습니다.
* **Hibernate 쓰기 지연에 따른 Unique 제약조건 충돌 버그 수정**:
  * 임시 유저 삭제와 기존 유저의 UUID 업데이트를 한 트랜잭션 내에서 처리할 때, Hibernate의 SQL 실행 우선순위 규칙(Update가 Delete보다 항상 먼저 실행됨)으로 인해 Unique 제약조건 충돌(`Duplicate entry` 에러)이 발생했습니다.
  * 이를 해결하기 위해 **삭제하기 전에 임시 유저의 `deviceUuid`를 `null`로 우선 갱신한 후 `saveAndFlush()`를 호출하여 점유를 즉시 해제**하고, 이후 `delete()`를 호출하여 `flush()` 처리를 즉시 완료함으로써 쿼리 실행 순서가 꼬여 발생하던 DB 트랜잭션 롤백 오류를 완벽하게 차단했습니다.

## 2. AI 식재료 보관 팁 검색 기능의 안정성 및 UX 개선

### 2.1. 구글 Gemini API 503/429 장애 예외 처리
* **친절한 에러 핸들링**: 구글 API의 일시적인 과부하로 인해 `503 Service Unavailable` 또는 `429 Too Many Requests` 상태가 반환되면, 백엔드가 500 내부 서버 에러를 던지는 대신 **"AI 보관법 안내 서비스에 일시적으로 많은 요청이 몰려 조회가 어렵습니다. 잠시 후 다시 시도해 주세요."**라는 명확한 원인 안내를 담아 `IllegalStateException` (400 Bad Request)으로 정제하여 클라이언트에 제공합니다.
* **백엔드 자동 재시도 (Auto-Retry) 로직 추가**: 일시적인 트래픽 폭주(Spike)를 극복하기 위해 구글 API 요청 실패 시 **점진적 대기 시간(1.5초 ➡️ 3초)을 두고 최대 3회까지 자동으로 재시도**를 수행하여 결과 수신 성공률을 대폭 향상시켰습니다.

### 2.2. 프론트엔드 UI/UX 사용성 개선
* **AI 안내 팝업 오버레이 구현**:
  * 식재료 보관 팁 모달에 처음 진입할 때 AI 분석 정보임을 고지하고 참고용으로 사용해 달라는 **안내 팝업창**을 오버레이 형태로 띄웁니다.
* **'일주일 동안 보지 않기' 기능 도입**:
  * 팝업 하단 왼쪽에 체크박스를 두어, 사용자가 일주일 동안 보지 않기를 선택하고 확인을 누르면 `AsyncStorage`를 통해 시각을 기억(`@hide_guide_notice_until`)하고, 일주일 동안 팝업 노출을 스킵해 주는 편리한 UX를 적용했습니다. (가독성 향상을 위해 본문 폰트 크기 `14`, 줄 간격 `22`로 조정)
* **네이티브 에러 Alert 창 적용**:
  * 백엔드 API가 최종 실패했을 때 단순히 화면 아래에 에러 텍스트만 띄우는 것이 아닌, 모바일 네이티브 Alert 팝업창(`Alert.alert`)을 띄워 사용자에게 현재 트래픽 초과 등의 구체적인 상황을 직관적으로 인식시킵니다.

### 2.3. Gemini 프롬프트 스크립트 고도화 (Prompt Engineering)
* **전문가 페르소나 적용**: 식품공학 박사 및 신선식품 보관 전문가 페르소나를 부여해 정확성 증대.
* **상세 기입 표준화**: `세척 유무`, `구체적인 포장 및 보관 장소`, `권장 보관 기간`, `온도/가스 민감성 및 주의점`을 필수로 구조화하여 작성하도록 지침을 세분화했습니다.
* **식재료명 정규화**: 사용자가 오타나 약어(예: 메론)로 검색하더라도 표준 한글 명칭(예: 멜론)으로 정규화하여 DB에 일관성 있게 축적하도록 개선했습니다.
